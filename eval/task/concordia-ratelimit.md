Concordia に「セッション別 提案レートリミット」機能を追加してください。

## 背景
Concordia は session ごとに AI セッションへ提案 (proposal) を dispatch します。暴走や過剰提案を
抑えるため、session ごとに「一定時間窓内の提案回数上限」を設けたい。既存には chat-mute /
rules-enabled などの runtime kill switch があり、schema_meta に永続化して Rules ページの GUI から
切り替えられます。

## 要件
1. **設定**: 「窓 (秒) と上限回数」を持つ。グローバル既定（例: 60 秒 / 5 回）＋ session 個別の上書き。
2. **適用**: dispatcher が提案を出す前にレートを確認し、窓内で上限に達していれば**今回の提案をスキップ**
   （理由をログ）。窓が明けたらカウントをリセットして再開する。
3. **永続化**: 設定（とカウンタ状態）を永続化する。既存の schema_meta / kill-switch / 設定永続化の
   作法に沿うこと。
4. **API**: グローバル既定と session 別設定を read / write する API を追加（既存の管制・設定
   エンドポイントの作法に合わせる）。
5. **UI**: Rules ページ等の管理画面からレートリミット設定を編集できるようにする。
6. **テスト**: 既存テストが通ること。レート制限の挙動（上限到達でスキップ／窓明けで再開）に最小限の
   テストを足すこと。

## 制約
- このリポジトリの**既存の規約・パターン・モジュール構成**に従うこと。新しい仕組みを別途作るのではなく、
  既存の kill-switch / 設定永続化 / dispatcher のゲート挿入の作法に寄せること。
- 1 つの自己完結した変更にまとめる（フェーズ分割しない）。
- 破壊的な DB 操作や無関係なリファクタはしない。

実装し、変更をコミットせずワーキングツリーに残してください（採点は diff で行います）。
