Concordia に「セッション一時停止 (paused)」機能を追加してください。

## 背景
Concordia は複数 AI セッションを協調・記録するサービスです。すでに chat-mute / rules-enabled /
proposer-interval などの **runtime kill switch** を持ち、schema_meta に永続化して Rules ページの
GUI から切り替えられます。

## 要件
1. 各セッションに boolean の `paused` 状態を持たせる（既定 `false`）。`paused` のセッションは
   提案・通知の対象から外れる、という意味づけ。
2. `paused` を read / write する API を追加する（既存の管制・設定エンドポイントの作法に合わせる）。
3. `paused` を永続化する（プロセス再起動後も保持）。
4. 既存の管理 UI（Rules ページ等のトグルがあるところ）から `paused` を切り替えられるようにする。
5. 既存テストが通ること。新規挙動には最小限のテストを足すこと。

## 制約
- このリポジトリの **既存の規約・パターン・モジュール構成に従う**こと。新しい仕組みを別途作るのではなく、
  既存の kill-switch / 設定永続化の作法に寄せること。
- 1 つの自己完結した変更にまとめる（フェーズ分割しない）。
- 破壊的な DB 操作や無関係なリファクタはしない。

実装し、変更をコミットせずワーキングツリーに残してください（採点は diff で行います）。
